HugoTinkerWorkflowTaylor

Workflow Process Concept of post/blog(anyone involved/creative team) Draft of written content/addign tags to categorize post(writers, with use of the true function on hugo) Dates for content to be published on(using hugo publishDate feature by publisher) Create the ability to see previously updated info(using lastmod by writers) Writing of core info(writers) Making sure core info is correct and ready to post(editors) Add an expiration date for post staying on main page (expiryDate-hugo/Publisher) Move to archives(if needed) (aliases-hugo) Some helpful additions to this workflow would be the use of microsoft word or a similar text editor, and a communication/organzation software such as slack or basecamps to help keep everyone on the same page.

Zach Schell Workflow

This was made in html Steps for a blog post- Idea in someone’s head for a post (anyone) > Creative team for approval to continue (all of creative team) > Author Assigned (Blog Manager) > Writing a rough draft (Author) > Rough draft edit (Editor) > Pith to creative team to continue (Author) > Revise draft (Author) > Edit draft (Editor) > Final Edit (Editor) > Write initial blog post (Web Designer) > Send final edit of post to blog manager (Editor) > Approves final draft (Blog Manager) > Send final script for upload (web designer) > Upload post to the local server (Web Designer) > Pitch full post to team (Author) > Published post on website (Web Designer) Responsibilities- I could see steps seven and eight being looped back between the author and the editors to get the post worded properly before the post could be published to the site.
